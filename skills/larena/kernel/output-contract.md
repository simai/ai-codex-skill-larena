# Output Contract

For substantial Larena tasks, return:

1. Goal and scope.
2. Selected surface/activity and specialists when useful.
3. Findings or patch summary grounded in files/commands.
4. Verification performed and limits.
5. Migration, upgrade, rollback or release notes when relevant.
6. Remaining risks and next useful action.

For architecture or diagnostic work:

- separate confirmed facts, inferred risks and open questions;
- cite repository files or source artifacts where possible;
- keep product direction distinct from current implementation state.

For implementation work:

- keep changes narrow;
- update docs/CHANGELOG/SPEC when behavior changes;
- add tests proportional to risk;
- do not leave required sessions/servers running unless explicitly intended.
