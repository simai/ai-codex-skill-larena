# Кэширование и производительность

## Базовые правила
- Сначала измеряй, потом оптимизируй.
- Фиксируй baseline (`до`) и результат (`после`) в отчете.
- Не вводи кэш без четкой стратегии инвалидации.

## Типовые оптимизации
- Убирай N+1 через eager loading и пересборку query.
- Добавляй индексы для hot-path фильтров/joins/sort.
- Убирай лишние сериализации payload для admin/API ответов.

## Simai-фокус
- Для EntityConfig ограничивай `allowedFields`, `includes`, `filters`.
- Не делай permissive сортировки/фильтры по тяжелым полям без индекса.
- При изменении админ-списков проверяй пагинацию и сортировку на больших выборках.

## Обязательные проверки
- Подтверждено отсутствие новых N+1 (query log/profiler).
- Есть измеримый эффект (время/queries/CPU).
- Регрессионные тесты или smoke-проверки не деградировали.
