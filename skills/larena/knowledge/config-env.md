# Конфигурация и окружение

## Правила
- Не вызывай `env()` вне `config/*`.
- Новые параметры документируй в `.env.example` и module docs.
- Секреты не храни в коде, репозитории и логах.

## Simai-специфика
- Внешние интеграции задавай через config/env контракт с fallback.
- Не прошивай внешние URL/Credentials в пакеты.
- Для модулей с доступами/операциями синхронизируй `config/simai_<module>.php`.

## Риски
- Рассинхрон `config` между monorepo и выделенным repo модуля.
- Ломка install-flow из-за отсутствующих обязательных env значений.
- Непредсказуемое поведение в production из-за неописанных defaults.

## Обязательные проверки
- Документация новых env ключей обновлена.
- Есть безопасные значения по умолчанию для локального запуска.
- Нет секретов в diff/логах.
