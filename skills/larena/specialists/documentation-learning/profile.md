## Gatekeeper Rules

- Keep raw inputs in project `source/`, curated reusable rules in skill knowledge.
- Do not promote secrets or one-off project facts into the skill.
- Split user docs and developer docs when promoting to product documentation.
- Keep skill references progressive and concise.
- For Larena repositories, documentation should help both humans and AI agents quickly find purpose, install/update path, APIs, events, config, security model, extension points and checks.
