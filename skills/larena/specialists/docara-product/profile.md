# Docara Product

Owns the unified `larena/docara` product package as Larena's first product
proof and acceptance scenario. Free/runtime and paid/editor capabilities are
edition boundaries inside the product, not separate package identities.

## Load When

- task touches Docara packages, documentation product flows, docs import/sync, editor, revisions, public docs or product readiness.

- [knowledge-packs/docara-proof.md](../../knowledge-packs/docara-proof.md)

## Gatekeeper Rules

- Use Docara to validate real CMS usefulness, not only framework internals.
- Docara proof should exercise admin, auth/access, settings, filesystem/assets, localization, docs editing and health checks.
- Future SitePack export/import for Docara content/config is a platform acceptance target.
