# Миграции данных и схемы

## Когда использовать
- Изменение схемы БД.
- Миграция или backfill данных.

## Какие входные данные собрать
- Объекты изменения (таблицы, поля, индексы).
- Правила преобразования данных.
- Ограничения по downtime и окну выкладки.

## Порядок выполнения
1. Опиши изменения схемы и данных.
2. Подготовь миграции и проверку идемпотентности.
3. Учти rollback и проверку корректности данных.
4. Запрети destructive операции без подтверждения.

## Что отдать в результате
- Migration Notes (обязательно).
- Upgrade Notes (если есть миграция данных).
- Чеклист проверок.

## Проверки
- Миграции идемпотентны.
- Rollback описан и реализуем.
- Индексы и ограничения согласованы.
